# C4 — Dispute Classifier Development

Work on the C4 LLM-powered Dispute Classification component.

## Architecture
Multi-stage dispute detection: keyword prefilter → negation detection → LLM classification
- **Prefilter**: Fast keyword matching (multilingual) — eliminates 95%+ of non-dispute events
- **Negation**: Detects negated dispute language ("NOT disputed", "dispute resolved")
- **LLM Backend**: Fine-tuned or hosted model for final classification
- **Taxonomy**: 5-class output: NO_DISPUTE, DISPUTE_POSSIBLE, DISPUTE_CONFIRMED, DISPUTE_RESOLVED, AMBIGUOUS

## Hard Block Behavior
DISPUTE_CONFIRMED or DISPUTE_POSSIBLE → pipeline BLOCKS bridge loan issuance.
This is a regulatory safety mechanism — no override available.

## Performance Targets
- False Negative rate: ≤ 2% (current: 8%)
- Prefilter latency: ≤ 2ms
- Full classification: ≤ 50ms (with LLM)

## LLM Backend Options
1. **Local quantized** — PEFT/LoRA fine-tuned model (GPU required)
2. **GitHub Models** — Free inference API (dev/staging)
3. **Groq** — Fast inference (production fallback)
4. **Any OpenAI-compatible** — Via openai SDK
