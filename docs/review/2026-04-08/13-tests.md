# Batch 13 — Tests

**Scope**: `lip/tests/` — 121 files, ~2,143 test functions, 303 mock usages across 30 files.
**HEAD**: `2b32314`
**Reviewer focus**: Mocking discipline (prior user feedback on mocked-DB pain), label-leakage guards, regression coverage for the findings raised in Batches 1-12, EPG-19 BLOCK-code drift test, fee floor invariants under `python -O`, flake risk from wall-clock dependencies, property-based test coverage.

## Summary

Test density is high — 2,143 test functions across 110 test files is strong for a codebase this size, and the test suite is honest about its infrastructure posture (`test_e2e_pipeline.py` is 100% in-memory, `test_e2e_live.py` gates on Redpanda via `@pytest.mark.live`, flaky SLO test is documented in CLAUDE.md). `conftest.py` provides deterministic `MockC1Engine` / `MockC2Engine` stubs so pipeline tests don't re-run ML inference. The test salt is a real 32-byte literal (not the zero-salt footgun). `test_fee_arithmetic.py` does QUANT spot-checks for the $57.53 / $575.34 canonical amounts. `test_c6_aml.py` verifies entity-ID hashing doesn't leak raw IDs.

The gaps are regression coverage for the findings raised in Batches 1-12 — **none of the Critical or High findings from earlier batches have regression tests**:

- **No pickle-load-rejection test** for C1/C2 (B10-01/02 RCE primitives). Zero files grep for `pickle` in the test tree.
- **No cross-language BLOCK-code drift regression** for the 6-way EPG-19 enumeration (B5-01/B6-01/B8-03). Seven files *use* BLOCK codes but none of them asserts set equality across Python / Rust / Go sources.
- **No fee floor test under `python -O`** for B10-08 (the `assert` that gets stripped). `test_fee_arithmetic.py` tests the normal path only.
- **No DP composition test** for B8-01 (the anonymizer charging 1ε for 3 noised values). `test_p10_anonymizer.py` exists but does not assert the composition-theorem budget accounting.
- **No TOCTOU race test** for B3-01 / query metering. No concurrent-thread torture test of the `check → record` boundary.
- **`conftest.py:38` hardcodes `THRESHOLD = 0.110`** as the C1 τ*. If C1 retrains with a new calibrated threshold, the pipeline fixtures silently use the stale value and every pipeline test passes against the wrong threshold. There is no assertion that the fixture threshold matches the current model artifact.
- **Only one file uses `hypothesis`** (`test_c1_inference_types.py`) — property-based testing is essentially absent from a codebase where it would pay off enormously (fee math monotonicity, velocity cap invariants, cascade probability bounds, rolling-window semantics under concurrent writes).
- **Wall-clock dependencies in 15 files** (`time.sleep`, `datetime.now`, `asyncio.sleep`) — flake candidates under CI load. The `test_slo_p99_94ms` flake is already documented in CLAUDE.md and the Q1 failures across C6 velocity / kill switch / e2e settlement are the same pattern.

## Notable positives (preserve)

- **2,143 test functions across 110 files** is strong test density for a codebase this size.
- **`conftest.py` is in-memory only** — E2E pipeline tests run without infrastructure, which is exactly the posture CLAUDE.md requires for safe local iteration.
- **Live vs unit separation is explicit** — `@pytest.mark.live` on `test_e2e_live.py` auto-skips when Redpanda is down. Clean dev UX.
- **`test_fee_arithmetic.py` does QUANT spot-checks** with the canonical $57.53 / $575.34 figures from the architecture spec. Any fee-math refactor that breaks these exact values fails loudly.
- **`test_c6_aml.py` verifies privacy invariants** — raw entity IDs never appear in hashed output. This is the right shape for a compliance-critical test.
- **`test_negation_suite.py` has 500 negation cases** across 5 categories — the right regression surface for C4's language understanding. CLAUDE.md's C4 rules depend on this file.
- **`conftest.py` MockC1/MockC2 stubs are deterministic** — no ML inference during pipeline tests. Correct separation of concerns.
- **Test salt is a real 32-byte literal** — not the zero-salt footgun from B10-07.
- **`test_e2e_pipeline.py` is explicitly documented as safe-without-infra** in CLAUDE.md — zero surprise during local runs.
- **Honest flake disclosure** — CLAUDE.md admits `test_slo_p99_94ms` is a timing flake and tells reviewers to ignore CPU-loaded failures. This is the right posture.
- **`test_dgen_adversarial.py` exists** — DGEN has adversarial input testing even though the broader DGEN review (B11-02) found label correctness issues. The adversarial test surface is the right foundation to build on.
- **`test_kill_switch_rust.py` exists** — the Rust FFI kill-switch has Python-level tests, closing the language boundary.
