# C3 — Repayment Engine Development

Work on the C3 Settlement Monitoring and Auto-Repayment component.

## Architecture
UETR-based settlement tracking + automatic repayment on confirmation + corridor buffers
- **Repayment Loop**: Polls SWIFT gpi UETR tracker for settlement confirmation
- **Settlement Handlers**: Process settlement events (ACSP, PART, RJCT)
- **UETR Mapping**: Maps payment UETRs to active bridge loans
- **Corridor Buffer**: Risk buffers by currency corridor type
- **Rejection Taxonomy**: Classifies rejection codes into maturity classes

## Rejection Code Classes → Maturity Windows
| Class | Maturity | Examples |
|-------|----------|---------|
| CLASS_A | 3 days | Technical failures, format errors |
| CLASS_B | 7 days | Account issues, insufficient funds |
| CLASS_C | 21 days | Compliance holds, sanctions review |
| BLOCK | 0 days | Fraud, sanctions match — NO bridge loan |

## State Machine
Loan: PENDING → APPROVED → DISBURSED → MONITORING → REPAID | DEFAULTED | RECOVERED
