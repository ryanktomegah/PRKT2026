# Batch 2 — API surface (`lip/api/`)

**Scope**: 13 files, ~3.1k LOC. FastAPI app, HMAC auth, admin/portfolio/cascade/regulatory/MIPLO routers, in-memory rate limiter.
**HEAD**: `2b32314`
**Reviewer focus**: authn/z defaults, signature scope, request validation, DoS surface, info-leak, multi-replica correctness.

## Summary

The API surface is functional but has **two security defaults that turn off the controls when an environment variable is missing** — this is the highest-impact class of finding in the batch. The HMAC scheme is correctly built (constant-time compare, replay window) but signs a too-narrow set of fields and is bypassable by simply not setting `LIP_API_HMAC_KEY`. The regulatory router enforces a differential-privacy budget with a documented TOCTOU race that violates DP soundness for any concurrent caller. The admin router exposes a `/model-card` endpoint that fabricates "is_synthetic=True" training results server-side and emits a regulatory model card from them — the same Delve fabrication pattern the integrity shield is supposed to prevent.

## Findings

| ID | Sev | File:line | Category | Finding | Suggested fix |
|---|---|---|---|---|---|
| B2-01 | **Critical** | `lip/api/app.py:108-113` | Auth disabled by default | `auth_dep = None` unless `LIP_API_HMAC_KEY` is set in the environment. All admin/portfolio/known-entities routers are wired with `dependencies=[Depends(auth_dep)] if auth_dep else []` (see `admin_router.py:155`). **Default state on a fresh deployment is fully unauthenticated.** A K8s deploy that forgets the env var (or rotates incorrectly) exposes platform-wide loan exposure, licensee IDs, total principal, and the model-card endpoint to anyone who reaches port 8080. | Make HMAC enforcement mandatory in non-dev mode. Add `LIP_API_REQUIRE_AUTH=true` default; if `LIP_API_HMAC_KEY` is missing under that flag, **refuse to start**. Dev/test override: `LIP_API_REQUIRE_AUTH=false` explicit. Defaults must fail closed. |
| B2-02 | **Critical** | `lip/api/admin_router.py:245-263` | Fabricated regulatory artifact | `/admin/model-card` accepts `component` as a query param and **forges** `model_metadata={"component": component}, training_results={"is_synthetic": True}, evaluation_results={"metrics": {}}` server-side. The endpoint then emits an authoritative-looking SR 11-7 model card from those forged inputs. *Conflicts with REX rule "will refuse to mark a model deployment-ready without a data card and out-of-time validation record"* and is exactly the Delve pattern `lip/integrity/` was built to prevent. | Either: (a) require the caller to supply the actual `model_metadata`/`training_results`/`evaluation_results` from the live `RegulatoryReporter` and validate they exist; or (b) delete the endpoint entirely. A model card endpoint that synthesizes its own inputs is a fraud surface, not a feature. |
| B2-03 | **Critical** (DP soundness) | `lip/api/regulatory_router.py:213-234` | TOCTOU on differential-privacy budget | Pre-check (`assert_within_budget`) and finalize (`record_query`) bracket the request handler. Comment: "Another concurrent request can consume the remaining budget between pre-check and finalize write." When that happens, the query has **already executed and returned data** under a budget that the metering refused to commit. Differential privacy guarantees rely on strict ε accounting; allowing concurrent over-spend is mathematically equivalent to no DP for those queries. The product literally sells a regulatory privacy guarantee here. | Move budget *consumption* to the pre-check (atomic compare-and-decrement against Redis-backed counter or in-process lock), not the finalize. Finalize only records latency/billing, not ε. Add a regression test that fires N concurrent queries near budget exhaustion and asserts the executed count never exceeds the budget. |
| B2-04 | High | `lip/api/auth.py:53-74` | HMAC scope too narrow | Signature covers `timestamp\|method\|path\|body` only. **Query string is not signed** (FastAPI's `request.url.path` strips it). **Host is not signed** — same key reused on staging+prod allows cross-host replay. **No signing-key identity / kid** — silent key rotations hide replays. No replay cache → within the 5-min window the same request can be replayed N times. | Sign `timestamp\|host\|method\|full_url_with_query\|key_id\|sha256(body)`. Maintain a per-key (timestamp, digest) replay cache for the 5-minute window. Add a `kid` header field and reject unknown ids fast. |
| B2-05 | High | `lip/api/app.py:112` | Length-dependent key decoding (footgun) | `hmac_key = hmac_key_hex.encode() if len(hmac_key_hex) < 64 else bytes.fromhex(hmac_key_hex)`. A 64-character key is hex-decoded; a 32-char key is used as raw UTF-8. Operators expecting hex who set a 32-char key get a different key than they think. No minimum length check — a 1-byte key is accepted. A 64-char non-hex string crashes the boot. | Require an explicit format: `LIP_API_HMAC_KEY_HEX` (64 chars exact, hex-decoded) or refuse. Enforce minimum 32 bytes. Validate at startup with a clear error. |
| B2-06 | High | `lip/api/app.py:115-124` | Silent crypto downgrade | `LIP_REGULATOR_SUBSCRIPTION_KEY_HEX` falls back to **raw UTF-8 encoding** if the value is not valid hex, with only `logger.warning(...)`. The regulator subscription tokens then get signed with a fundamentally different key than the rotator intended. Operator never sees the warning in time → tokens fail verification later for "no reason." | Fail-fast: if the hex parse fails, raise at startup. Same applies to B2-05. |
| B2-07 | High | `lip/api/regulatory_router.py:60-78` | Rate limiter keyed on wrong identity | `_make_rate_limit_dep` uses `request.client.host` (peer IP). Behind any L7 proxy / ingress / load balancer, every request appears to come from the proxy IP → **all regulators share one bucket**. The actual rate limit collapses to global. The regulator subscription token already exists at this layer; key on `token.regulator_id` instead. | Move the rate limiter dep below the regulator-token dep, and key the bucket on `token.regulator_id`. If you must key on IP, parse `X-Forwarded-For` against an explicit trusted-proxy list. |
| B2-08 | High | `lip/api/app.py:174-179` | `/metrics` exposed unauthenticated | "Prometheus /metrics scraping endpoint — no auth required". Prometheus metrics frequently leak label cardinality (per-licensee loan counts, corridor distributions). At low loan counts, label cardinality can identify individual licensees. Standard practice is basic-auth or network-policy restriction. | Require either basic auth (Prometheus supports it) or restrict via NetworkPolicy to the in-cluster Prometheus pod CIDR. Document the threat model in code, not just in this review. |
| B2-09 | High | `lip/api/admin_router.py:217-242` | Unbounded compute on POST | `POST /admin/stress-test` accepts `num_simulations: int = 1000` with **no upper bound**. A caller (or a misconfigured automation) can pass `num_simulations=10_000_000` → CPU pinned, request thread blocked, /metrics scraping degraded. Authenticated DoS. | Cap with `Query(default=1000, ge=10, le=10_000)`. Run anything larger as an async job, not in the request thread. |
| B2-10 | High | `lip/api/auth.py:69` | Future-dated tokens accepted | `abs(now - request_ts) > 300` is symmetric — a request timestamped 4:59 in the future is accepted. Pre-signing tokens with future timestamps is a classic abuse vector (sign once, replay over the entire forward window). | Use an asymmetric window: forward ≤ 30 s (clock skew), backward ≤ 300 s. Reject `request_ts > now + 30`. |
| B2-11 | Medium | `lip/api/rate_limiter.py:14-83` | Per-pod, not global | In-memory bucket. With N replicas, each licensee can do N× the documented rate. Same `_redis` anti-pattern as B1-02 (here it's not even a stub — there's no Redis hook at all). | Back the bucket store with Redis (`INCR + EXPIRE` or a Lua script for token-bucket math). At minimum, document loudly that "N replicas → N× rate" so operators don't oversize the documented limit. |
| B2-12 | Medium | `lip/api/auth.py:99-129` | Silent absence on missing FastAPI | The `make_hmac_dependency` factory is wrapped in `try: from fastapi import …` … `except ImportError: pass`. If FastAPI isn't installed, the function silently doesn't exist — callers get `NameError` instead of `ImportError`. | Define a stub that raises `ImportError` with a clear message, mirroring the `app.py` pattern at line 273. |
| B2-13 | Medium | `lip/api/app.py:62` | Silent setup, dropped reference | `BorrowerRegistry(redis_client=redis_client)` is constructed and immediately discarded — the comment "init loads from Redis; shared via C7 config" is wishful. The object is GC'd before any C7 component sees it. Same pattern as B1-02. | Either store on `application.state` for downstream access, or remove the line. As-is it does nothing useful and confuses readers. |
| B2-14 | Medium | `lip/api/regulatory_router.py:121-156` | Body double-read risk | `_extract_corridors_from_request` calls `await request.body()` for stress-test. Starlette caches the body, so this is OK today, but the path is fragile — any future Starlette change to streaming bodies breaks both the auth-corridor check and FastAPI's pydantic body parse. Also: a malicious large body is parsed twice (once for corridor extraction, once by pydantic). | Have FastAPI parse the body once (use a `Depends` that reads `StressTestRequest` then runs the corridor permission check with the parsed model). | B2-15 | Medium | `lip/api/regulatory_router.py:425` | Domain leak via exception coercion | `except ValueError: raise HTTPException(404, "Report not found")` — any `ValueError` from anywhere in `render_report()` becomes 404. | B2-17 | Medium | `lip/api/regulatory_router.py:485-528` | Authorization bypass on no-token | `/usage/{regulator_id}` only enforces ownership *if* a Bearer token is present (`if auth_header.startswith("Bearer ")`). ## Notable positives (preserve)

- `auth.py` correctly uses `verify_hmac_sha256` (constant-time compare) and includes a 5-minute replay window. The overall shape of the HMAC scheme is the right pattern, even if the scope (B2-04) is too narrow.
- `regulatory_router.py` `_make_rate_limit_dep` uses `check_and_consume_with_remaining` (single lock acquisition, no TOCTOU between `remaining()` and `check_and_consume()`) — exactly the right primitive.
- The factory pattern (`make_*_router(service, ...)`) keeps service objects testable without an HTTP server. Avoid breaking this in future routers.
- `app.py` `lifespan` cleanly drains notification service and closes Redis on shutdown.
- `regulatory_router.py` uses pydantic models with explicit `pattern=` and `ge=/le=` bounds on every query param — good FastAPI hygiene.
