# Batch 3 — C8 License Manager (`lip/c8_license_manager/`)

**Scope**: 5 files, ~1.1k LOC. License token issuance/verify, boot validator + kill-switch wiring, regulator subscription tokens, query/revenue metering.
**HEAD**: `2b32314`

## Summary

The license-token shape is defensible — HMAC-SHA256 over a canonical sorted-keys JSON, constant-time compare, expiry check, kill-switch on boot failure. Two real problems: (1) the **query metering primitive has the same TOCTOU race as B2-03** at the function level — `assert_within_budget` releases its lock between read and check, so concurrent regulators can all pass the pre-check, and the post-record path increments unconditionally; this is the *underlying* primitive that B2-03 calls into, so fixing B2-03 must fix this file; (2) the **canonical payload is hand-maintained** — adding any new field to `LicenseToken` requires also updating `canonical_payload`, `to_dict`, `from_dict`, and `sign_token`; a missed update silently leaves the new field unsigned, which is a privilege-escalation primitive in a token system. The metering store is also purely in-memory: a redeploy resets every regulator's monthly budget.

## Notable positives (preserve)

- `verify_token` uses `hmac.compare_digest` (constant-time).
- `boot_validator.py` is **fail-closed** by default — missing token, missing key, or any parse/HMAC error all engage the kill switch. This is the right shape for a license gate; preserve it through any refactor.
- Boot validator hard-checks `aml_dollar_cap_usd == _AML_CAP_UNSET` before construction — even though the sentinel reachability is broken (B3-03), the *intent* is right and well-commented.
- Processor licensee BIC validation is regex-strict (`^[A-Z0-9]{8}([A-Z0-9]{3})?$`) and refuses on any non-match.
- `regulator_subscription.py` `from_dict` validates `valid_until > valid_from`, `query_budget_monthly > 0`, `privacy_budget_allocation > 0` — good upfront input validation.
- `LicenseBootValidator.context` is a property, not a settable attribute — caller cannot mutate post-validation.
