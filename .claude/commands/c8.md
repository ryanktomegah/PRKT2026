# C8 — License Manager Development

Work on the C8 Technology Licensing Enforcement component.

## Architecture
HMAC-SHA256 token-based license enforcement for BPI technology licensing model
- **License Token**: Cryptographic token encoding licensee_id, expiry, feature flags
- **Boot Validator**: Prevents unlicensed component startup
- **TPS Cap**: Per-licensee transaction throughput cap

## Key Files
| File | Purpose |
|------|---------|
| `lip/c8_license_manager/__init__.py` | Public API |
| `lip/c8_license_manager/license_token.py` | Token generation + validation (HMAC-SHA256) |
| `lip/c8_license_manager/boot_validator.py` | Startup license check |

## Business Model
BPI is a technology licensor. Banks deploy LIP and pay:
- **Platform royalty**: 15% of fee_repaid_usd per bridge loan
- Licensed via time-limited HMAC tokens with feature flags
- `licensee_id` flows through C7 decision log for royalty calculation

## Token Structure
```
{
  "licensee_id": "BANK_ABC_001",
  "issued_at": "2026-01-01T00:00:00Z",
  "expires_at": "2027-01-01T00:00:00Z",
  "features": ["c1", "c2", "c3", "c4", "c5", "c6", "c7"],
  "tps_cap": 1000,
  "hmac": "<HMAC-SHA256 signature>"
}
```

## Enforcement Points
1. **Boot**: C7 agent calls `validate_license()` at startup — refuses to start if invalid
2. **Runtime**: `licensee_id` injected into every pipeline result
3. **TPS cap**: Enforced at C5 Kafka consumer level

## Security Rules
- Token signing key is NEVER stored in code or config files
- Signing key is provided via environment variable or KMS
- Token tampering is detected via HMAC verification
- Expired tokens are rejected — no grace period

## Canonical Constants
- PLATFORM_ROYALTY_RATE: 15% (Decimal("0.15"))
