# C6 — AML Velocity Development

Work on the C6 Anti-Money Laundering and Velocity Monitoring component.

## Architecture
Sanctions screening + velocity limits + anomaly detection + cross-licensee salt rotation
- **Sanctions Loader**: Fetches and parses OFAC SDN + EU consolidated lists
- **AML Checker**: Orchestrates all AML checks for a payment event
- **Velocity Engine**: Rolling 24hr windows per entity (dollar cap + txn count)
- **Anomaly Detection**: Beneficiary concentration, unusual patterns
- **Cross-Licensee**: Salted hashing for privacy-preserving cross-bank intelligence
- **Salt Rotation**: 365-day rotation cycle with 30-day overlap

## Hard Block Behavior
Sanctions match OR velocity breach → pipeline BLOCKS bridge loan. No override.

## Velocity Limits
- Dollar cap: $1,000,000 per entity per 24hr rolling window
- Transaction count cap: 100 per entity per 24hr
- Beneficiary concentration: >80% to single beneficiary triggers alert

## Security Rules (CIPHER domain)
- NEVER commit `c6_corpus_*.json` (typology patterns)
- Entity pseudonymization: SHA-256 only (NOT MD5)
- Salt rotation: 365 days, 30-day overlap for transition
- Cross-licensee salts are unique per licensee pair

## Sanctions Sources
- OFAC SDN List (US Treasury)
- EU Consolidated Financial Sanctions List
- Updated weekly via `update-sanctions.yml` GitHub Action
