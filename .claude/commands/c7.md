# C7 — Execution Agent Development

Work on the C7 Bridge Loan Execution Agent component.

## Architecture
Bank-side execution agent with safety controls and audit logging
- **Agent**: Receives LoanOffer from pipeline, executes bridge loan lifecycle
- **Kill Switch**: Emergency stop — halts all loan issuance instantly
- **Human Override**: Manual approval/rejection for flagged loans
- **Degraded Mode**: Operates with reduced functionality when services are down
- **Decision Log**: Immutable audit trail for every decision (7-year retention)

## Three-Entity Model
C7 runs as the **ELO** (Execution Lending Organisation) — the bank-side agent.
- **MLO**: Capital provider
- **MIPLO**: Platform operator (BPI)
- **ELO**: Bank-side agent executing loans against its own book

## Safety Controls
1. **Kill Switch**: Any authorized user can halt all issuance. Requires explicit re-enable.
2. **Human Override**: Loans above threshold or flagged by C4/C6 require manual approval.
3. **Degraded Mode**: If C4 or C6 are unavailable, agent enters conservative mode (higher thresholds).
4. **Decision Log**: Every decision (approve, reject, override, kill) is logged with timestamp, actor, reason.

## C8 Integration
C7 checks C8 license token at boot — refuses to start without valid license.
`licensee_id` is attached to every decision log entry for audit trail.
