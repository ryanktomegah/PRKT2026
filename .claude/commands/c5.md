# C5 — Streaming Infrastructure Development

Work on the C5 Real-Time Event Ingestion and Normalization component.

## Architecture
Kafka consumer → event normalization → pipeline dispatch
- **Kafka Worker**: Consumes pacs.002 events from SWIFT message queue
- **Event Normalizer**: Transforms raw ISO 20022 XML/JSON into NormalizedEvent
- **Flink Jobs**: Stream processing for aggregations and windowed analytics
- **Redis Config**: State store for velocity windows and deduplication

## Key Files
| File | Purpose |
|------|---------|
| `lip/c5_streaming/__init__.py` | Public API |

## NormalizedEvent Schema
Key fields: uetr, sending_bic, receiving_bic, currency_pair, amount_usd,
payment_status, rejection_code, hour_of_day, settlement_lag_days,
prior_rejections_30d, data_quality_score, correspondent_depth, message_priority

## Infrastructure Dependencies
- **Apache Kafka**: Message broker (SWIFT pacs.002 events)
- **Apache Flink**: Stream processing (windowed aggregations)
- **Redis**: State store (velocity windows, UETR dedup, session state)

## Rules
- E2E tests require live Kafka/Redis — excluded from local CI
- Kafka consumer uses at-least-once delivery — idempotency via UETR dedup
- Redis TTLs must match UETR_TTL_BUFFER_DAYS (45 days)
